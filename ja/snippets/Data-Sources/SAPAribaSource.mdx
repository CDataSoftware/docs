## Prerequisites

Before you can connect SAP Ariba Sourcing, you must create a custom OAuth app as follows:

<Steps>
<Step>
Go to the [SAP Developer Portal](https://developer.ariba.com/) and sign in.
</Step>
<Step>
Click **Manage** (bottom icon on the left).
</Step>
<Step>
Click the **+** button next to the search field.
</Step>
<Step>
Fill in **Application Name** and **Description**, and then click **Submit**.
</Step>
<Step>
Click **Actions** > **Request API Access**.
</Step>
<Step>
Select the API you want to retrieve data from. This corresponds to the API connection property.
</Step>
<Step>
Select your realm.
</Step>
<Step>
Click **Submit**. Wait until the request is approved.
</Step>
<Step>
Once approved, click **Actions** > **Generate OAuth Secret**.
</Step>
<Step>
Save the values displayed for **API key**, **OAuth Client Id**, and **OAuth Client Secret**.
</Step>
</Steps>

## セットアップガイド

以下の手順で、SAP Ariba Sourcing をConnect AI アカウントに接続します。

<Steps>
<Step>
Connect AI のダッシュボードから**Connections** ページを開きます。
</Step>
<Step>
右上の **+ Add Connection** をクリックします。
</Step>
<Step>
検索フィールドに *SAP Ariba Sourcing* と入力し、データソース名をクリックします。
</Step>
<Step>
新しいコネクションの**Basic Settings** タブで、コネクション名を入力するかデフォルトの名前をそのまま使用します。
</Step>
<Step>
Enter the following information:
	- **API Key**—the API key created in the prerequisites.
	- **Realm**—the realm from which you want to access data.
	- **API**—the API you want to retrieve data from.
	- **Data Center**—the data center where your account's data is hosted.
	- **Type Detection Scheme**—select whether to return all columns as the string type (None) or scan rows to determine the data type (RowScan).
	- **Include Custom Fields**—toggle on to include custom fields from SAP Ariba tables.
	- **OAuth Client Id**—the OAuth client Id from the prerequisites.
	- **OAuth Client Secret**—the OAuth client secret from the prerequisites.
	- **Max Threads**—the maximum number of concurrent requests the provider can execute to SAP Ariba.
	- **Environment**—the environment to use when connecting to SAP Ariba, either PRODUCTION or TEST.
</Step>

</Steps>
